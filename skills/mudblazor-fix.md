# MudBlazor Safety Fix — $ARGUMENTS

## ACTIVATION RULE

Run this skill automatically after BrandSync MCP generates any MudBlazor UI code.
Do not skip it — the fixes are not optional, they prevent silent rendering failures.

Triggers:
- BrandSync Pocket 3 pipeline completes (Razor or CSS files written)
- User asks to build/add/create any UI in this MudBlazor project
- Any `var(--bs-*)` token is written to a CSS file
- Any `<MudIcon>` is added to a Razor file

Fix MudBlazor code generated by BrandSync MCP. Catches the class of bugs that happen
when generated code assumes tokens or CSS patterns that don't work in Blazor.

---

## WHY THIS SKILL EXISTS

BrandSync MCP generates correct-looking code that silently breaks in MudBlazor because:

1. **Token drift** — MCP emits `var(--bs-surface-container)` but tokens.css may only have
   `--bs-surface-base`. CSS var() fails silently: backgrounds become transparent, spacing
   collapses to 0, icons blow up.

2. **MudIcon SVG expansion** — MudIcon renders as `<span class="mud-icon-root"><svg viewBox="0 0 24 24"/></span>`.
   SVG with no explicit width/height in a flex container expands to fill available space.
   The CSS reset `* { box-sizing: border-box }` makes this worse.

3. **MudBlazor specificity** — `mud-*` class overrides lose to MudBlazor's own selectors
   without `!important`.

4. **Component class names** — MudBlazor renders components to specific HTML class names
   (`.mud-button-root`, `.mud-chip-root`, `.mud-table-root`) that differ from what MCP assumes.

## STEP 3 — Scan Razor Files for MudBlazor Issues

Read the generated Razor file(s). Check for:

### MudIcon without scoped size constraint
Pattern: `<MudIcon` anywhere in the file.
If found → CSS MUST include the MudIcon lock (see STEP 4).

### MudButton without custom class
```razor
❌ <MudButton Variant="Variant.Filled" Color="Color.Primary">
✓  <MudButton Variant="Variant.Filled" Color="Color.Primary" Class="btn-mud btn-primary-mud">
```

### MudChip/badge without custom class
```razor
❌ <MudChip Color="Color.Success">Active</MudChip>
✓  <MudChip Color="Color.Success" Class="badge-mud badge-success-mud" T="string">Active</MudChip>
```

### Scope wrapper missing
Every page MUST be wrapped in `.eg-dashboard-scope` (set on the layout or the page):
```razor
<div class="eg-dashboard-scope">...</div>
```

## STEP 4 — Apply Fixes

### FIX A: MudIcon Lock (add to top of scoped CSS, always)

```css
/* Lock MudIcon SVGs — prevents flex-stretch blowup */
.eg-dashboard-scope .mud-icon-root {
    width: 20px !important;
    height: 20px !important;
    font-size: 20px !important;
    flex-shrink: 0 !important;
    display: inline-flex !important;
    align-items: center !important;
    justify-content: center !important;
    overflow: hidden !important;
    min-width: 20px !important;
    min-height: 20px !important;
    max-width: 20px !important;
    max-height: 20px !important;
}
.eg-dashboard-scope .mud-icon-root svg {
    width: 20px !important;
    height: 20px !important;
    min-width: 20px !important;
    min-height: 20px !important;
    max-width: 20px !important;
    max-height: 20px !important;
    display: block !important;
}
```

### FIX C: Add !important to mud-* overrides

Any rule that targets a MudBlazor class needs `!important`:

```css
/* ✗ Will lose to MudBlazor's own styles */
.eg-dashboard-scope .mud-button-root { background-color: var(--c-primary); }

/* ✓ Wins */
.eg-dashboard-scope .mud-button-root { background-color: var(--c-primary) !important; }
```

### FIX D: Replace broken tokens with scoped variables

Instead of scattering hardcoded hex values everywhere, define them as scoped variables
at the top of the CSS, then use the scoped variables:

```css
/* Define once at top of CSS */
.eg-dashboard-scope {
    --bg: #F9FAFB;           /* replaces var(--bs-surface-container) */
    --bg-white: #ffffff;     /* replaces var(--bs-surface-base) */
    --bg-hover: #EFF0F8;     /* replaces var(--bs-surface-hover) */
    --bg-active: #DEE2ED;    /* replaces var(--bs-surface-selected) */
    --text: #000000;
    --text-sec: #5D6472;
    --text-muted: #828998;
    --border: 1px solid #DEE2ED;
    --br: 8px;
    --t: 0.2s ease;
    /* Keep confirmed tokens as-is */
    /* --bs-color-primary-default, --bs-text-default, etc. still used directly */
}

/* Use scoped vars everywhere */
.eg-dashboard-scope .nav-item:hover { background-color: var(--bg-hover); }
```
